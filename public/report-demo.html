Critical Content-Security-Policy (CSP) Header Missing FAIL ‚Ä∫

Description

The Content-Security-Policy header is not set. CSP is the most powerful browser-side defense against Cross-Site Scripting (XSS) attacks, data injection, and unauthorized resource loading.

Impact

  • Attackers can inject malicious scripts that steal user data, session tokens, and credentials
  • No protection against inline script execution
  • Third-party resources can be loaded without restriction
  • Data exfiltration via injected content cannot be blocked

Risk Level

CRITICAL ‚Äî XSS is the #1 web vulnerability (OWASP Top 10). Without CSP, any XSS flaw becomes fully exploitable.

Remediation ‚Äî Cloudflare Dashboard

Go to Rules ‚Üí Transform Rules ‚Üí Modify Response Header and add:

Header: Content-Security-Policy
Value: default-src 'self'; script-src 'self' 'unsafe-inline' 'unsafe-eval'; style-src 'self' 'unsafe-inline'; img-src 'self' data: https:; font-src 'self' data:; connect-src 'self' https:; frame-ancestors 'none';

Remediation ‚Äî Next.js (next.config.js)

// next.config.js
const securityHeaders = [
  {
    key: 'Content-Security-Policy',
    value: [
      "default-src 'self'",
      "script-src 'self' 'unsafe-inline' 'unsafe-eval'",
      "style-src 'self' 'unsafe-inline'",
      "img-src 'self' data: https:",
      "font-src 'self' data:",
      "connect-src 'self' https:",
      "frame-ancestors 'none'"
    ].join('; ')
  }
];

module.exports = {
  async headers() {
    return [{ source: '/(.*)',
              headers: securityHeaders }];
  }
};

‚ö° Quick Start

Start with a permissive policy (above) and tighten over time. Use Content-Security-Policy-Report-Only first to test without breaking your site.

Critical X-Frame-Options Header Missing FAIL ‚Ä∫

Description

The X-Frame-Options header is not present. This header prevents your site from being embedded in iframes on malicious websites.

Impact

  • Clickjacking attacks: Attackers overlay invisible iframes of your site, tricking users into clicking buttons/links they can't see
  • Users could unknowingly authorize payments, change settings, or leak data
  • Particularly dangerous for sites with authenticated actions

Remediation ‚Äî Cloudflare

Rules ‚Üí Transform Rules ‚Üí Modify Response Header:

Header: X-Frame-Options
Value: DENY

Use SAMEORIGIN instead if you embed your own site in iframes.

Remediation ‚Äî Nginx

add_header X-Frame-Options "DENY" always;
Critical HTTP Strict Transport Security (HSTS) Missing FAIL ‚Ä∫

Description

HSTS is not configured. This header forces browsers to always use HTTPS, preventing SSL-stripping and man-in-the-middle downgrade attacks.

Impact

  • First-time visitors can be intercepted on HTTP before redirect
  • Vulnerable to SSL-stripping attacks on public WiFi
  • Cookie theft via unencrypted initial connection

Remediation ‚Äî Cloudflare Dashboard

Go to SSL/TLS ‚Üí Edge Certificates ‚Üí Enable HSTS:

  • Status: ON
  • Max-Age: 12 months (31536000)
  • Include subdomains: Yes
  • Preload: Yes
  • No-Sniff: Yes

Remediation ‚Äî Manual Header

Strict-Transport-Security: max-age=31536000; includeSubDomains; preload

After confirming HTTPS works everywhere, submit to hstspreload.org for browser preload list inclusion.

High DMARC Record Missing FAIL ‚Ä∫

Description

No DMARC (Domain-based Message Authentication, Reporting & Conformance) record found at _dmarc.scoreforai.com. DMARC ties SPF and DKIM together and tells receiving servers what to do with unauthenticated email.

Impact

  • Attackers can send phishing emails that appear to come from @scoreforai.com
  • No visibility into email authentication failures
  • Damages domain reputation and email deliverability
  • Fails compliance requirements (Google/Yahoo 2024 sender requirements)

Remediation ‚Äî DNS Record

Add a TXT record in your Cloudflare DNS:

Name: _dmarc
Type: TXT
Value: v=DMARC1; p=none; rua=mailto:dmarc@scoreforai.com; ruf=mailto:dmarc@scoreforai.com; fo=1;

Start with p=none (monitor mode), then after reviewing reports, escalate to p=quarantine and eventually p=reject.

Progressive Enforcement

  • Week 1-4: p=none ‚Äî collect reports
  • Week 5-8: p=quarantine; pct=25 ‚Äî quarantine 25%
  • Week 9+: p=reject ‚Äî full enforcement
High Referrer-Policy Header Missing FAIL ‚Ä∫

Description

The Referrer-Policy header is not set. Without it, the browser's default behavior sends the full URL (including query parameters) to third-party sites.

Impact

  • Sensitive data in URLs (tokens, user IDs) leaked to third parties
  • Internal URL structure exposed to external analytics

Remediation

Referrer-Policy: strict-origin-when-cross-origin

Medium Permissions-Policy Header Missing FAIL ‚Ä∫

Description

The Permissions-Policy (formerly Feature-Policy) header is not set. This header controls which browser features and APIs can be used on your site.

Impact

  • Embedded content could access camera, microphone, geolocation
  • Third-party scripts can use powerful browser APIs without restriction

Remediation

Permissions-Policy: camera=(), microphone=(), geolocation=(), interest-cohort=()

This disables camera, microphone, geolocation, and FLoC/Topics API for all contexts. Add back specific permissions as needed.

Medium X-Content-Type-Options Header Missing FAIL ‚Ä∫

Description

The X-Content-Type-Options: nosniff header is missing. This prevents browsers from MIME-type sniffing, which can turn non-executable MIME types into executable ones.

Impact

  • Uploaded files could be executed as scripts by the browser
  • MIME confusion attacks possible

Remediation

X-Content-Type-Options: nosniff

Single value, no configuration needed. This is enabled automatically when you turn on HSTS with "No-Sniff" in Cloudflare.

Medium DKIM Not Found WARN ‚Ä∫

Description

No DKIM records were found for common selectors (default, google, selector1, selector2, k1, mail, dkim). DKIM cryptographically signs outgoing emails to prove they haven't been tampered with.

Impact

  • Emails may fail authentication at strict receivers
  • Required for DMARC enforcement
  • Reduced email deliverability

Remediation

If using Cloudflare Email Routing, DKIM is handled automatically. If using a third-party email provider, add their DKIM CNAME/TXT records to your DNS. Check with your email provider for specific instructions.

All-in-One Next.js Fix

Add all security headers at once in your next.config.js. Here's the complete configuration:

// next.config.js ‚Äî Complete Security Headers
const securityHeaders = [
  { key: 'Strict-Transport-Security', value: 'max-age=31536000; includeSubDomains; preload' },
  { key: 'X-Frame-Options', value: 'DENY' },
  { key: 'X-Content-Type-Options', value: 'nosniff' },
  { key: 'Referrer-Policy', value: 'strict-origin-when-cross-origin' },
  { key: 'Permissions-Policy', value: 'camera=(), microphone=(), geolocation=(), interest-cohort=()' },
  { key: 'Content-Security-Policy', value: "default-src 'self'; script-src 'self' 'unsafe-inline' 'unsafe-eval'; style-src 'self' 'unsafe-inline'; img-src 'self' data: https:; font-src 'self' data:; connect-src 'self' https:; frame-ancestors 'none';" },
];

module.exports = {
  poweredByHeader: false,
  async headers() {
    return [{ source: '/(.*)', headers: securityHeaders }];
