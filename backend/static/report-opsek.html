Critical Content-Security-Policy (CSP) Header Missing FAIL ‚Ä∫

Description

The Content-Security-Policy header is not set. CSP is the most powerful browser-side defense against Cross-Site Scripting (XSS) attacks, data injection, and unauthorized resource loading.

Impact

  • Attackers can inject malicious scripts that steal user data, session tokens, and credentials
  • No protection against inline script execution
  • Third-party resources can be loaded without restriction
  • Data exfiltration via injected content cannot be blocked

Risk Level

CRITICAL ‚Äî XSS is the #1 web vulnerability (OWASP Top 10). Without CSP, any XSS flaw becomes fully exploitable.

Remediation ‚Äî vercel.json

Add security headers directly in your Vercel project configuration:

{
  "headers": [
    {
      "source": "/(.*)",
      "headers": [
        {
          "key": "Content-Security-Policy",
          "value": "default-src 'self'; script-src 'self' 'unsafe-inline' 'unsafe-eval'; style-src 'self' 'unsafe-inline'; img-src 'self' data: https:; font-src 'self' data:; connect-src 'self' https:; frame-ancestors 'none';"
        }
      ]
    }
  ]
}

Remediation ‚Äî next.config.js

// next.config.js
const cspHeader = [
  "default-src 'self'",
  "script-src 'self' 'unsafe-inline' 'unsafe-eval'",
  "style-src 'self' 'unsafe-inline'",
  "img-src 'self' data: https:",
  "font-src 'self' data:",
  "connect-src 'self' https:",
  "frame-ancestors 'none'"
].join('; ');

module.exports = {
  async headers() {
    return [{
      source: '/(.*)',
      headers: [{
        key: 'Content-Security-Policy',
        value: cspHeader
      }]
    }];
  }
};

‚ö° Quick Start

Start with Content-Security-Policy-Report-Only first to test without breaking your site. Monitor the browser console for violations, then switch to enforcing mode.

Critical X-Frame-Options Header Missing FAIL ‚Ä∫

Description

The X-Frame-Options header is not present. This header prevents your site from being embedded in iframes on malicious websites, making it vulnerable to clickjacking attacks.

Impact

  • Clickjacking attacks: Attackers overlay invisible iframes of your site, tricking users into clicking buttons/links they can't see
  • Users could unknowingly authorize payments, change settings, or leak data
  • Particularly dangerous for sites with authenticated actions

Remediation ‚Äî vercel.json

{
  "key": "X-Frame-Options",
  "value": "DENY"
}

Use SAMEORIGIN instead if you embed your own site in iframes.

Remediation ‚Äî next.config.js

{ key: 'X-Frame-Options', value: 'DENY' }

Add this to the security headers array (see complete config in Action Plan below).

Medium X-Content-Type-Options Header Missing FAIL ‚Ä∫

Description

The X-Content-Type-Options: nosniff header is missing. This prevents browsers from MIME-type sniffing, which can turn non-executable MIME types into executable ones.

Impact

  • Uploaded files could be executed as scripts by the browser
  • MIME confusion attacks possible

Remediation

{ key: 'X-Content-Type-Options', value: 'nosniff' }

Single value, no configuration needed. Add to your vercel.json or next.config.js headers array.

High Referrer-Policy Header Missing FAIL ‚Ä∫

Description

The Referrer-Policy header is not set. Without it, the browser's default behavior sends the full URL (including query parameters) to third-party sites when users click outbound links.

Impact

  • Sensitive data in URLs (tokens, user IDs) leaked to third parties
  • Internal URL structure exposed to external analytics

Remediation

{ key: 'Referrer-Policy', value: 'strict-origin-when-cross-origin' }

Add to your vercel.json or next.config.js headers array.

Medium Permissions-Policy Header Missing FAIL ‚Ä∫

Description

The Permissions-Policy (formerly Feature-Policy) header is not set. This header controls which browser features and APIs can be used on your site.

Impact

  • Embedded content could access camera, microphone, geolocation
  • Third-party scripts can use powerful browser APIs without restriction

Remediation

{ key: 'Permissions-Policy', value: 'camera=(), microphone=(), geolocation=(), interest-cohort=()' }

This disables camera, microphone, geolocation, and FLoC/Topics API for all contexts. Add back specific permissions as needed.

High DMARC Record Missing FAIL ‚Ä∫

Description

No DMARC (Domain-based Message Authentication, Reporting & Conformance) record found at _dmarc.opsek.io. DMARC ties your existing SPF and DKIM together and tells receiving servers what to do with unauthenticated email.

Impact

  • Attackers can send phishing emails that appear to come from @opsek.io
  • No visibility into email authentication failures
  • Damages domain reputation and email deliverability
  • Fails Google/Yahoo 2024 sender requirements for bulk senders

Remediation ‚Äî DNS Record

Add a TXT record in your DNS provider:

Name: _dmarc
Type: TXT
Value: v=DMARC1; p=none; rua=mailto:dmarc@opsek.io; ruf=mailto:dmarc@opsek.io; fo=1;

Since you already have SPF and DKIM (Google Workspace) configured, DMARC will immediately start providing value by reporting authentication results.

Progressive Enforcement

  • Week 1-4: p=none ‚Äî collect reports, verify Google Workspace alignment
  • Week 5-8: p=quarantine; pct=25 ‚Äî quarantine 25% of failing mail
  • Week 9+: p=reject ‚Äî full enforcement

Google Workspace tip: Use a free DMARC report analyzer like Postmark DMARC to parse the XML reports Google sends.

Add all security headers at once. Choose either next.config.js (recommended) or vercel.json:

Option A ‚Äî next.config.js (Recommended)

// next.config.js ‚Äî Complete Security Headers for opsek.io
